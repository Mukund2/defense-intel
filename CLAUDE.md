# Defense Intel - Project Instructions

## Mission

Read `MISSION.md` at the start of every session. Every feature, every decision filters through one question: **does this help an underdog win a contract they deserve?** If it doesn't, don't build it.

## Collaboration Style

- **Be incredibly hands-on with the user.** This is a deeply collaborative project — don't go off and build things solo.
- **Use AskUserQuestion frequently** during implementation and brainstorming. Ask about design decisions, UX choices, feature priorities, naming, architecture tradeoffs — anything where the user's opinion matters.
- **Don't assume — ask.** When there are multiple reasonable approaches, present them and ask which direction to go. The user wants to be in the loop on decisions, not just see finished output.
- **Use deep web searches liberally.** When researching APIs, data sources, competitors, defense contracting patterns, or anything where real-world data would be more accurate than guessing or asking the user, do the research first. Prefer WebSearch/WebFetch over asking the user factual questions you could look up.
- **Ask the user for opinions and preferences, not for facts.** If it's googleable, google it. If it's a judgment call, ask.
